## [2026-04-15] 7주차: 리스트 렌더링 심화 및 컴포넌트 순수성

### 1. 개요 및 이론
- **순수 함수(Pure Function)의 조건**:
    - **자신의 일에만 집중**: 호출되기 전에 존재했던 객체나 변수를 변경하지 않음.
    - **같은 입력, 같은 결과**: 동일한 입력이 주어지면 항상 동일한 JSX를 반환함.
- **사이드 이펙트(Side Effect)**: 렌더링 과정과 직접적인 관련이 없는 외부 상태 변경(변수 수정, 데이터 업데이트 등)을 의미하며, 리액트 컴포넌트는 이를 피해야 함.
- **지역 변경(Local Mutation)**: 렌더링 도중 내부에서 생성된 변수를 수정하는 것은 순수성을 유지하므로 안전함.
- **Key Prop**: 리스트 항목의 고유 식별자로, 데이터의 `id`를 사용하여 리액트의 렌더링 효율을 높임.

### 2. 실습 내용: 리스트 필터링 및 Fragment 적용
- **데이터 필터링**: `filter()`와 `map()`을 조합하여 조건에 맞는 데이터만 동적으로 렌더링하는 구조 설계.
- **Fragment Key**: 여러 태그를 반환할 때 `<Fragment>`에 `key`를 할당하여 리스트 렌더링 에러 해결.

### 3. 실습 내용: 순수성 유지 및 에러 핸들링
- **Strict Mode 테스트**: 외부 변수를 수정하는 비순수 컴포넌트가 두 번 호출될 때 발생하는 버그를 통해 순수성 유지의 중요성 체득.
- **TeaGathering 실습**: 반복문을 사용해 함수 내부 배열에 데이터를 추가(Local Mutation)하여 사이드 이펙트 없이 다수의 컴포넌트를 생성하는 실무 기법 습닥.

## [2026-04-08] 6주차: 조건부 및 리스트 렌더링 숙달

### 1. 개요 및 이론
- **논리 연산자 AND(&&)**: `if`문이나 삼항 연산자 외에 자주 사용되는 방식으로, 조건이 `true`일 때만 우측의 JSX를 렌더링함.
    - **주의사항**: 왼쪽 조건이 숫자 `0`일 경우, React는 `0` 자체를 렌더링하므로 `messageCount > 0 && ...` 처럼 명확한 부울(bool) 조건을 만드는 것이 중요함.
- **변수에 조건부 JSX 할당**: `let` 변수에 기본 내용을 먼저 담고, `if`문으로 조건에 따라 JSX를 재할당한 뒤 최종 변수를 반환하는 방식. 코드가 복잡할 때 가독성이 높음.
- **리스트 렌더링과 Key**: `map()` 함수로 리스트를 생성할 때, 각 항목은 고유한 `key` prop을 가져야 함.
    - **중요성**: React는 이 `key`를 통해 어떤 항목이 변경, 추가, 삭제되었는지 식별하여 렌더링 성능을 최적화함. `key`가 없으면 콘솔에 "Each child in a list should have a unique 'key' prop" 경고가 발생함.

### 2. 실습 내용: 여행 짐 챙기기 (PackingList)
- **Step 1~4**: `PackingList`와 `Items` 컴포넌트 생성 및 기초 `props` 전달 구조 설계.
- **Step 5~6**: `if`문을 이용한 기본적인 조건부 렌더링(체크 표시 추가) 구현.
- **Step 7~8**: 삼항 연산자를 활용한 코드 최적화 및 `<del>` 태그를 이용한 가독성 개선 실습.
- **Step 9**: `let` 변수(`itemContent`)에 조건부로 JSX를 할당하여 렌더링 로직을 정교하게 분리하는 심화 실습 진행.

### 3. 실습 내용: 영화 영웅 리스트 (MovieHeroes)
- **Step 1**: 영화 영웅 데이터가 담긴 `heroes` 배열을 선언하고, `map()` 함수를 사용하여 `<li>` 태그 리스트를 동적으로 출력하는 컴포넌트 구현.
- **에러 핸들링**: 리스트 출력 시 발생하는 **Key Prop 경고 메시지**를 확인하고, 리액트에서 고유 식별자(`key`)가 필요한 이유와 해결 방안 학습.

## [2026-04-01] 5주차: Props 데이터 흐름 및 JSX 마크업 정밀 규칙

### 1. 개요 및 이론
- **Props의 정의와 특징**: 
  - 부모 컴포넌트가 자식 컴포넌트에게 전달하는 '데이터 꾸러미'입니다.
  - **단방향 통신**: 데이터는 언제나 부모에서 자식으로만 흐릅니다.
  - **읽기 전용**: 자식 컴포넌트는 받은 Props를 직접 수정할 수 없습니다.
- **JSX 3가지 핵심 규칙**:
  1. 하나의 루트 엘리먼트로 반환할 것 (Fragment `<></>` 활용).
  2. 모든 태그는 반드시 닫아줄 것.
  3. 속성(Attribute)은 **카멜 케이스(camelCase)**로 작성할 것 (예: `tabIndex`, `strokeWidth`).
- **용어 구분**: 태그(Tag), 엘리먼트(Element), 어트리뷰트(Attribute), 프로퍼티(Property)의 개념적 차이 학습.

### 2. 실습 내용
- **객체 형태의 Props 전달**: 개별 변수가 아닌 `imageInfo` 같은 객체 형태로 데이터를 묶어 전달하여 컴포넌트의 재사용성을 높이는 실습 진행.
- **Props 기본값(Default) 설정**: 부모로부터 전달받은 데이터가 없을 경우를 대비해 변수 선언부에서 기본값을 지정하는 방법 숙달.
- **JSX Spread 문법**: `{...props}`를 사용하여 여러 속성을 한 번에 전달하는 효율적인 방법 학습 및 가독성 저하 등의 주의사항 확인.

## [2026-03-25] 4주차: 효율적인 모듈 관리 및 JSX의 본질 이해

### 1. 개요 및 이론
- **Named Export 권장 이유**: 
  - 팀원 간 일관성 있는 컴포넌트 명명 가능.
  - 에디터 기능을 통한 안전한 이름 변경(리팩토링) 용이성.
  - **트리 쉐이킹(Tree Shaking)**: 빌드 시 사용하지 않는 코드를 제거하여 최적화에 유리.
- **JSX의 개념**: JavaScript를 확장한 문법으로, UI 로직과 마크업을 동기화된 상태로 안전하게 관리하기 위해 사용함.

### 2. 실습 내용
- 다양한 Export/Import 방식을 실습하고, 컴포넌트와 파일 이름을 의미 있게 명명하는 습관 체득.
- Web 인터랙티브 환경에 맞춰 로직이 내용을 결정하는 구조적 설계 실습.

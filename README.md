### 2. 이벤트 전파(Event Bubbling)와 제어 (3.2)
- **이벤트 전파(Bubbling) 원리**: DOM 트리의 하위 엘리먼트(`button`)에서 발생한 이벤트가 상위 부모 엘리먼트(`nav`, `div` 등)로 순차적으로 전달되는 현상임. 
- **전파 중지 (`e.stopPropagation()`)**: 핸들러의 매개변수인 이벤트 객체(`e`)를 통해 전파를 막을 수 있음. 이를 통해 부모 컴포넌트의 핸들러가 의도치 않게 실행되는 것을 방지함.
- **React의 전파 예외**: React에서 `onScroll`을 제외한 거의 모든 이벤트는 전파됨. `onScroll`은 정의된 JSX 태그 내에서만 실행됨.

### 3. 브라우저 기본 동작 방지 (3.2.3)
- **기본 동작의 예**: HTML의 `<form>` 태그는 제출(`onSubmit`) 시 페이지 전체를 새로고침(Reload)하는 기본 동작을 가지고 있음.
- **방지 방법 (`e.preventDefault()`)**: 이벤트 핸들러 내부에서 `e.preventDefault()`를 호출하여 브라우저의 고유 기능을 실행하지 않도록 설정할 수 있음.
- **전파 중지 vs 동작 방지**: `stopPropagation()`은 상위 태그로의 전달을 막는 것이고, `preventDefault()`는 특정 이벤트에 할당된 브라우저의 기본 행동(Link 이동, Form 제출 등)만 막는 것임.

- **이벤트 핸들러와의 관계**: 핸들러는 순수 함수일 필요가 없으므로 사이드 이펙트(값 변경)를 처리하기에 최적이며, 여기서 **State Hook**을 호출해 정보를 저장하고 화면 갱신을 트리거함.

## [2026-05-06] 10주차: 이벤트 핸들러와 Props 연동

### 1. 이벤트 핸들러 함수의 전달 규칙
- **전달 vs 호출 구분 (중요)**: `onClick={handleClick}`처럼 함수 이름만 넘기면 클릭 시 실행되지만, `onClick={handleClick()}`처럼 소괄호를 붙이면 렌더링 시점에 즉시 실행되므로 주의가 필요함.
- **인라인 핸들러 주의사항**: `onClick={alert('...')}` 형태는 렌더링마다 실행되므로, `onClick={() => alert('...')}` 화살표 함수로 감싸야 클릭 시에만 실행됨.
- **React의 내부 처리**: `<button>` 태그도 React 내부에서 `React.createElement("button", { onClick: handleClick }, "버튼 클릭")` 형태로 변환되기 때문에 props를 전달할 수 있음.

### 2. 이벤트 핸들러에서 Prop 사용하기 (3.1.2)
- 이벤트 핸들러가 컴포넌트 내부에 선언되어 있으면 부모로부터 전달받은 **props에 클로저로 접근** 가능함.
- **children prop**: `<Button>버튼1</Button>`처럼 여는/닫는 태그 사이의 내용이 자동으로 `children` prop으로 전달됨. 직접 정의하지 않아도 태그 사이 내용을 그대로 넘길 수 있음.
- **`&nbsp;`**: JSX에서 버튼 사이 간격을 띄우기 위해 HTML 엔티티를 사용함.

### 3. 이벤트 핸들러를 Prop으로 전달하기 (3.1.3)
- 하나의 핸들러 함수만 사용하면 버튼마다 다른 동작을 구현하기 어렵기 때문에, **이벤트 핸들러 자체를 prop으로 전달**하는 방식을 사용함.
- 부모 컴포넌트(Toolbar)에서 자식 컴포넌트(Button)를 호출할 때 이벤트 핸들러를 함께 전달하고, 자식은 받은 핸들러를 그대로 실행함.

### 4. 실습 내용: 이벤트 핸들러 분리 및 동영상 재생/정지
- **Step 1 - handle.jsx 분리**: 이벤트 핸들러를 별도 파일로 분리하고 `export`하여 외부에서 가져다 쓸 수 있도록 모듈화함.
- **Step 2 - 핸들러를 prop으로 전달**: Toolbar에서 `handleClick`을 import하여 Button 컴포넌트의 `handle` prop으로 전달하고, Button은 받은 `handle`에 `message`를 인자로 넘겨 실행함.
- **Step 3 - 동영상 재생/정지 구현**: `handlePlay`, `handleStop` 함수를 추가하고, `<video id="videoPlayer">` 태그를 렌더링한 뒤 `message` prop으로 id를 전달하여 DOM을 직접 조작함.
    - `document.getElementById(id)`: HTML 문서에서 고유한 id를 가진 요소를 찾아 JavaScript 객체로 반환하는 메서드. 요소가 없으면 `null`을 반환함.
    - `handlePlay({ message })` → `document.getElementById(message).play()`
    - `handleStop({ message })` → `document.getElementById(message).pause()`

### 1. 개요 및 이론
- **순수 함수(Pure Function)의 조건**:
    - **자신의 일에만 집중**: 호출되기 전에 존재했던 객체나 변수를 변경하지 않음.

- **사이드 이펙트(Side Effect)**: 렌더링 과정과 직접적인 관련이 없는 외부 상태 변경(변수 수정, 데이터 업데이트 등)을 의미하며, 리액트 컴포넌트는 이를 피해야 함.
- **지역 변경(Local Mutation)**: 렌더링 도중 내부에서 생성된 변수를 수정하는 것은 순수성을 유지하므로 안전함.
- **Key Prop**: 리스트 항목의 고유 식별자로, 데이터의 `id`를 사용하여 리액트의 렌더링 효율을 높임.

  1. 하나의 루트 엘리먼트로 반환할 것 (Fragment `<></>` 활용).
  2. 모든 태그는 반드시 닫아줄 것.
  3. 속성(Attribute)은 **카멜 케이스(camelCase)**로 작성할 것 (예: `tabIndex`, `strokeWidth`).
- **용어 구분**: 태그(Tag), 엘리먼트(Element), 어트리뷰트(Attribute), 프로퍼티(Property)의 개념적 차이 학습.

### 2. 실습 내용
- **객체 형태의 Props 전달**: 개별 변수가 아닌 `imageInfo` 같은 객체 형태로 데이터를 묶어 전달하여 컴포넌트의 재사용성을 높이는 실습 진행.
- **Props 기본값(Default) 설정**: 부모로부터 전달받은 데이터가 없을 경우를 대비해 변수 선언부에서 기본값을 지정하는 방법 숙달.
- **JSX Spread 문법**: `{...props}`를 사용하여 여러 속성을 한 번에 전달하는 효율적인 방법 학습 및 가독성 저하 등의 주의사항 확인.

## [2026-03-25] 4주차: 효율적인 모듈 관리 및 JSX의 본질 이해

### 1. 개요 및 이론
- **Named Export 권장 이유**: 
  - 팀원 간 일관성 있는 컴포넌트 명명 가능.

- **JSX의 개념**: JavaScript를 확장한 문법으로, UI 로직과 마크업을 동기화된 상태로 안전하게 관리하기 위해 사용함.

### 2. 실습 내용
- 다양한 Export/Import 방식을 실습하고, 컴포넌트와 파일 이름을 의미 있게 명명하는 습관 체득.
- Web 인터랙티브 환경에 맞춰 로직이 내용을 결정하는 구조적 설계 실습.
